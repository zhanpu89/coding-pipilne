# 评审报告模板

按评审模式跳转对应模板。模板强制结构：一~五 + 变更记录，所有 `{...}` 占位符必须替换。

## 模板一：需求评审报告

- 头部：REV-REQ-YYYYMMDD-NNN
- 一、评审概要（结论/P0/P1/P2/是否允许进入架构设计）
- 二、问题清单（ID/等级/描述/影响/建议/阻断标记）
- 三、需求缺陷详情（P0展开/P1展开/P2改进建议）
- 四、评审建议
- 五、评审结论（✅/⚠️/❌ + 条件）
- 变更记录

## 模板二：架构评审报告

- 头部：REV-ARCH-YYYYMMDD-NNN
- 一、评审概要
- 二、问题清单
- 三、架构缺陷详情（技术选型/安全/可扩展性/单点故障）
- 四、评审建议
- 五、评审结论
- 变更记录

## 模板三：详细设计评审报告

- 头部：REV-DES-YYYYMMDD-NNN
- 一、评审概要
- 二、问题清单
- 三、设计的缺陷详情（并发/事务/幂等/异常/数据模型）
- 四、评审建议
- 五、评审结论
- 变更记录

## 模板四：测试用例评审报告

- 头部：REV-TST-YYYYMMDD-NNN
- 一、评审概要
- 二、问题清单
- 三、测试用例缺陷详情（覆盖度/场景穿透/异常覆盖）
- 四、评审建议
- 五、评审结论
- 变更记录

## 模板五：全流程追溯矩阵

- 头部：REV-TRC-YYYYMMDD-NNN
- 一、追溯矩阵表（需求→架构→详设→代码→测试，逐行标记状态 ✅/⚠️/❌/N/A）
- 二、未覆盖需求清单
- 三、断层分析
- 四、综合结论
