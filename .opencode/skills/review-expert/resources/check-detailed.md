# 详细设计评审检查清单

仅保留项目特有方法论。通用评审项（并发/事务/幂等/异常/数据模型/可观测性）AI 按标准实践执行。

## 3.7 功能链完整性（项目特有框架）

| 断裂类型 | 检查项 |
|---------|--------|
| 上游缺口 | 需求侧期望接口 vs 后端已规划接口，发现缺口记为 P0 |
| 链路缺失 | A→B→C 流程中 B 未实现 |
| 状态断裂 | 状态机缺少逆路径（如审核通过→驳回无实现） |
| 依赖缺失 | 某接口依赖的数据来源未定义 |
| 异常断裂 | 异常流程无兜底逻辑 |
| 数据断裂 | 接口返回字段在后续流程未被使用 → 确认是否遗漏 |
| 上游对齐 | 比对 `doc/arch/` 和 `doc/prd/` 接口完整性，缺失=P0 |

## 3.8 跨端对齐（仅多端）

- 逐端提取接口清单
- 存在性比对（缺失=P0）
- 字段类型对齐（不一致=P1）

信息缺失 >30% → ❌ 不通过。
