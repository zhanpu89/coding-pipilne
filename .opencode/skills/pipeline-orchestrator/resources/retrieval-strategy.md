# 检索策略（按需加载）

## 搜索方式选择

| 目标 | 参数 | 原因 |
|------|------|------|
| 精确函数名/报错信息 | `use_fts=true` | 精确匹配 |
| 模块内历史 | `module=模块名` | 最强过滤 |
| 标签明确的已知问题 | `tags=标签名` | 人工分类 |
| 模糊描述/跨模块 | `use_vector=true` | 语义匹配 |
| 以上都不确定 | 先用 FTS5，不足时再用向量 | 精确优先 |

## 搜索执行（多角度探索）

一轮探索：

```
角度 A — 模块名
  search_summaries(query=任务涉及的模块名, module=模块名, limit=3)
  如不确定模块名则跳过此角度。

角度 B — 技术关键词
  search_summaries(query=1~2 个具体技术词, use_fts=true, limit=3)
  提取原则：优先取函数名/类名/框架名/库名/报错信息。
  不要提取通用词如"修复""新增""问题"。

角度 C — 自然语言描述
  search_summaries(query=任务核心问题(20 字以内), use_vector=true, limit=3)
  use_vector=true 是语义检索，不需要关键词重叠。
  把任务的核心矛盾说出来即可。

补充兜底（以上相关结果合计 < 3 条时）：
  list_recent_sessions(limit=5)
```

## 相关性判断

对每条返回结果执行：

| 等级 | 判断依据 | 处理 |
|------|---------|------|
| P-高 | 模块相同 + 问题类型相似 | 必须注入上下文 |
| P-中 | 模块不同但技术栈/问题模式相同 | 有条件注入 |
| P-低 | 仅时间接近或关键词偶然匹配 | 跳过 |

## 注入格式

```
【历史经验参考（来自项目记忆）】
搜索角度 A（模块名: xxx）：
  - [{P-高/P-中}] {session title} — {summary 第一句}
搜索角度 B（技术关键词: xxx）：
  - [{P-高/P-中}] {session title} — {summary 第一句}
搜索角度 C（自然语言: xxx）：
  - [{P-高/P-中}] {session title} — {summary 第一句}
注入理由：
  - {结果1} 对本次任务有帮助：{具体说明}
  - {结果2} 对本次任务有帮助：{具体说明}
```

**注入前最后筛子：** 每条注入结果必须能回答"对下一步 Phase 有什么具体帮助？"。答不出就放弃注入，宁可少不可滥。
