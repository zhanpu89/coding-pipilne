# 代码评审报告模板

输出 `doc/review/{模块名}_代码评审报告{_RN}.md`。纯后端模式不含维度0和前端节。

## 报告结构

**头部：** CR-YYYYMMDD-NNN / 版本 / 状态 / 评审模式 / 日期 / 关联文档

**一、评审概要：** 结论(✅/⚠️/❌) / P0/P1/P2 数量 / 是否允许进入测试

**二、评审范围：** 后端文件清单 / 前端文件清单(全栈)

**三、问题清单：** ID / 等级 / 文件位置 / 问题描述 / 修复建议 / 阻断标记

**四、问题详情：**
- 维度0（全栈）：前后端契约一致性 — 后端覆盖率/前端字段一致性/小程序一致性/端间一致性
- 维度1~5：编码规范 / 业务逻辑 / 安全 / 性能 / 可维护性
- 维度6（全栈）：前端审查（编码/逻辑/安全/性能/可维护性/框架特化）

**五、修复建议：** P0 应如何修复 / P1 建议修复方案 / 改进优先级

**六、评审结论：** ✅ 无P0且P1≤2 | ⚠️ 无P0但P1>2 | ❌ 有P0
