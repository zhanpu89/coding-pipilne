# 代码评审检查清单（大纲）

语言无关通用层。具体检查项 AI 按标准工程实践执行。

## 维度 1：编码规范

- 命名惯例（语言驼峰/下划线/帕斯卡）
- 日志用框架非裸输出
- 统一响应格式（code + message + data）
- 语言特化内容见 `lang-ext.md`

## 维度 2：业务逻辑

- 业务规则完整，伪代码均已实现
- 接口契约与详设一致
- 写入操作有事务/原子性保护
- 异常按场景分类处理

## 维度 3：安全

- SQL注入 / NoSQL注入 / 命令注入
- 越权（水平+垂直），敏感信息不泄露
- 输入校验，幂等性保障

## 维度 4：性能

- 无 N+1 查询，无大事务
- 缓存穿透/击穿/雪崩防护
- 批量操作有节流

## 维度 5：可维护性

- 圈复杂度 ≤ 15，函数 ≤ 50 行
- 无重复代码（抽离复用）
- 无魔法数字，有类型定义
- 注释说明"为什么"而非"是什么"
