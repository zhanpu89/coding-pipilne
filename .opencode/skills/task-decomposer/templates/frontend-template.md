# 前端 / 小程序详细设计文档模板

按端跳转 `##` 节。配合 `resources/frontend-guide.md` 使用。

## Web 前端详细设计文档模板

每页面域一份，保存为 `前端_{页面域}.md`。

**编号：** FE-DES-YYYYMMDD-NNN，关联后端详设文档。

**结构：**

1. 页面清单（路由/权限/布局/说明）
2. 页面交互规则（初始化行为 / 操作与反馈 / 校验规则 / 空态错误态 / 状态变更）
3. API 调用表（接口路径 / 请求参数 / 响应字段 / 调用时机 — **逐字段与后端 OpenAPI 核对**）
4. 组件拆分（名称 / 职责 / 输入输出）
5. 数据模型（store 结构 / 缓存策略 / 本地存储）
6. 路由配置（路径 / 懒加载 / 守卫 / 参数）
7. 错误处理规范（全局 / 业务 / 网络各层兜底）
8. 交互与体验（骨架屏 / 防抖节流 / 乐观更新 / 确认弹窗 — LC-FE-001 实现分支：Vue3/React）
9. 可测试性（可 mock 边界 / 重点覆盖场景 / 快照范围）
10. 依赖关系
— 变更记录

## 微信小程序详细设计文档模板

每页面域一份，保存为 `小程序_{页面域}.md`。

**编号：** MP-DES-YYYYMMDD-NNN，关联后端详设文档。

**结构：**

1. 页面清单（路径 / 权限 / 分包归属 / 说明）
2. 交互规则（初始化 / 操作与反馈 / 空态错误态）
3. API 调用表（同 Web 前端 — **逐字段核对**）
4. 组件拆分
5. 数据模型（全局 vs 页面级 / 缓存）
6. 路由配置（Tab 页 / 非 Tab 页层级控制 / 参数传递）
7. 错误处理
8. 微信能力（登录 / 支付 / 分享 — 仅标记需要的能力，不重复描述）
9. 分包策略（主包 ≤2MB，按模块分包）
10. 可测试性
— 变更记录
