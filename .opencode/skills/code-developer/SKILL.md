## Step 0

确认 `doc/detailed/`、`doc/arch/tech-stack.json`、`doc/detailed/项目规则.md`、`doc/detailed/编码规范.md` 都存在。缺少任一则停止。确定实现顺序（从依赖底层开始）。

## Step 1

对每个详设文档，生成实体/DAO/Service/API/DTO + 单元测试骨架。每模块完成后按 `.opencode/rules/doc-alignment.md` 同步更新文档。

### 写入检查

- 所有 BR 有代码实现
- OpenAPI 端点有对应接口
- 异常处理覆盖安全要求
- 符合 LC/ER 约束
- 无 `TODO`/`FIXME` 残留

## 规则

- 详设为唯一来源，禁止偏离接口契约和业务规则
- 逐模块实现，写完一个再下一个
- 编码规范 > 框架默认 > 个人偏好
- 200 行能写成 50 行就重写。不实现未要求的功能
- 只改目标模块的文件，不改无关代码

## 熔断

缺少关键输入 / LC-001 无法识别 / 写入失败 → 停止
