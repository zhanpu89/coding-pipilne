# 测试报告模板

输出 `doc/tester/{模块名}_测试报告[_RN].md`

**头部：** TR-YYYYMMDD-NNN / 版本 / 状态 / 日期 / 关联用例

**一、执行概要：** 总计/通过/失败/阻塞/跳过 / 结论(✅/⚠️/❌)

**二、缺陷清单：** BUG-ID / 关联用例 / 描述 / 严重程度(P0-P3) / 状态

**三、测试详情：** 模块/类型/总数/通过/失败/阻塞/跳过

**四、P0 阻断分析（有则）：** 原因 / 影响 / 修复建议

**五、结论：** ✅ PASS≥90% | ⚠️ PASS≥80% | ❌ 有P0或PASS<80%

多轮时新增「上轮缺陷回归情况」节。
