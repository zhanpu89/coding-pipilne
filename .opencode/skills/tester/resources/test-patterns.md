# 测试模式与质量标准参考

## 通用原则

- 单元测试隔离外部依赖，只测业务逻辑
- 集成测试用测试数据库，验证完整流程
- 每业务规则至少 1 正 + 1 反用例
- 边界值：min/max/boundary±1

## 执行命令

| LC-001 | 命令 |
|--------|------|
| java | `mvn test -pl {模块} -Dtest={类名}` |
| python | `pytest tests/{模块}/ -v` |
| go | `go test ./{模块}/... -v` |
| nodejs | `npx jest --testPathPattern={模块}` |

## 覆盖率目标

单元 ≥80%，集成 ≥60%，关键路径 100%。

## 缺陷 ID

`BUG-{模块}-{序号}` + 关联用例 + 描述 + 严重程度(P0-P3)。P0 暂停测试。
