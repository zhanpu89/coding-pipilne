# 访谈与提问框架

Step 1-2 加载。5W1H + KANO + 冲突处理。

**5W1H：** Who(用户) What(问题) Why(业务驱动) When(时机) Where(场景) How(流程)

**KANO 分层：** Must-have(不做不通过) / Performance(越多越好) / Excitement(超出预期)

**冲突处理：** 3 次不一致 → 记录待定。用户连续 2 轮无法回答 → 记录"待补充"继续。

**范围确认：** 端类型/用户角色/主流程/关键异常/成功指标/MVP 边界 → 全部确认后进入 Step 3。

**端专属问点：** 后端→数据量/扩展性/集成方；Web→浏览器兼容/响应式/SEO；小程序→微信能力/分包/审核
