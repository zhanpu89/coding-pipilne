# PRD 填写指南

Step 3 加载。每节填写标准 AI 已知，只列关键约束。

- **执行摘要：** `[产品] 是 [用户] 用于 [功能] 的 [类型]，解决 [痛点]`
- **目标：** SMART（具体/可衡量/可实现/相关/有时限），M/S 级功能有 Given/When/Then AC
- **用户角色：** 角色名称+描述+使用频率+权限范围
- **功能需求：** M/S 级=AC(BDD格式)，C/W 级=简述
- **MoSCoW：** M(不通过无意义) / S(重要但可延后) / C(锦上添花) / W(本期不做)
- **NFR 基准：** Web 首屏 <3s，API P99 <500ms，并发=峰值*1.5
- **风险：** 每个 M/S 功能至少 1 个风险条目
- **AC 规则：** 禁止技术描述，用业务语言写 Given/When/Then
