# 端专属章节

按端类型跳转。通用章节来自 `templates/common.md`。

## 后端专属章节

组合：common.md 头部+第1-4节+第6-11节 + 本文件第5节（业务规则与数据规范）。

规则 R-004：后端 PRD 不包含第 5 节"用户界面需求"。

## Web 前端专属章节

组合：common.md 头部+第1-4节+第6-11节 + 本文件第5节（UI 与交互需求）。

规则 R-005：前端 PRD 只描述用户可见的交互行为（点击/跳转/输入），不描述技术实现。每项 UI 需求：
- 一句话描述页面/弹窗/组件的业务目的
- 元素的交互行为（非技术术语）
- 校验规则（非 SQL/API）

## 微信小程序专属章节

组合：common.md 头部+第1-4节+第6-11节 + 本文件第5节（UI 与微信能力）。

规则 R-006：小程序 PRD 描述微信能力时只写业务目的（"用户可通过微信登录"），不写 API 名称。登录/支付/分享每项需求含：微信能力名称（业务目的）+ 用户操作流程 + 期望结果。
